Data Tables

Markup

Tables have a number of optional classes. table is the default class that is applied to ALL data tables. table-striped gives every other row a slight gray background. table-hover gives a light gray hover on mouseover.

All table tags must have a caption as the first nested element for accessibility.

<table class="table table-striped table-hover">
    <caption>This is a brief description of the data in this table.</caption>
	...

The table headers receive scope="col" for accessibility reasons.

<thead>
  <tr>
    <th scope="col">&nbsp;</th>
    <th scope="col">Table Heading - Words</th>
    <th scope="col">Numbers with Decimals</th>
    <th scope="col">Whole Numbers</th>
    <th scope="col">Date</th>
  </tr>
</thead>

Table rows and cells don't receive any required markup or classes. They are dependent on proper structure within table class="table".

<tr>
  <td>
    <form action="" class="form-stacked">
      <div class="form-group">
        <div class="checkbox">
          <input type="checkbox" id="checkbox-td1" name="options" value="checkbox-td1">
          <label for="checkbox-td1">&nbsp;</label>
        </div>
      </div> 
    </form> 
  </td>
  <td>Table Cell A</td>
  <td>3.14</td>
  <td>8</td>
  <td>03/03/2014</td>
</tr>

Tables with a large number of headers (nearly all of our tables), will be wrapped in a div with a class of "table-wrap". To be safe, please wrap all tables in this div. 

<div class="table-wrap">
  ... table goes here ...
</div>
This wrapper will allow the table to scroll horizontally if and when it becomes wider than the module.
