Default Buttons

Description

It is preferred to use button tags over input or a tags for button elements. Each will function similarly, but for the sake of consistency, please use the button tag when possible. All buttons are given the .btn class and either .btn-default, or .btn-submit class. The .btn-submit class may only be used for 'final actions' within a user activity.

The .btn-submit class may only be used for 'final actions' within a user activity.

Buttons may utilize class flush to remove all margins. For instance, if a button needed to sit on the right side of a module and line up with other elements, this class would remove the 10px of right margin, allowing it to be positioned correctly.

Accessibility

Every element with a .btn class should also receive a role="button" attribute. For actual button elements this is redundant, but an input or anchor tag using this class will be more clear.

Example

Default Button

Markup:HTML

<a href="#" role="button" class="btn btn-default">Default Button</a>
<button href="#" role="button" class="btn btn-default">Default Button</button>
<input type="button" role="button" value="Default Button" class="btn btn-default">

Submit Buttons

Description

These style of buttons represent a final action in a task. There will never be more than one submit button on a page, or in a task.

Accessibility

Every element with a .btn class should also receive a role="button" attribute. For actual button elements this is redundant, but an input or anchor tag using this class will more clear.

Example

Submit

Markup:HTML

<a href="#" role="button" class="btn btn-submit">Submit</a> 
<button role="button" class="btn btn-submit">Submit</button>
<input role="button" type="button" value="Submit" class="btn btn-submit">

Cancel, Delete, Clear Buttons / Links

Description

These will typically be paired with Submit buttons. They receive their own classes in case we decide to style them differently in the future. All three classes receive the same CSS currently.

Example

Cancel
Discard
Clear

Accessibility

Each of these links is given a semantic class of link-buttonname as well as a rel value matching the text of the link.

Markup:HTML

<a href="#" rel="cancel" class="link-cancel">Cancel</a>
<a href="#" rel="discard" class="link-discard">Discard</a>
<a href="#" rel="clear" class="link-clear">Clear</a>
