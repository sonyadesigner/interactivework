Git - Code Review Process

1. Clone the Project

It is assumed that Git is installed, SSH Key sent to project lead and that you have navigated to the directory you wish to place Doug but have not yet began working on the framework.

$ git clone git@gitlab.tk20.com:dboehm/framework.git

2. Initialize Git

This will tell Git that this directory should be tracked.

$ cd framework

Moves into the project directory

$ git init
$ git add -A
$ git commit -m "initial commit"

3. Create a Local CodeReview Branch

This isn't entirely necessary, but will help keeping active work and codereview work separate.

$ git checkout -b codereview

4. Pull Down The CodeReview Branch

This grabs the code meant to be reviewed, it is likely the most current code, but still in development and may be broken. Notes will have to be provided to specifically mention which code is to be reviewed.

$ git pull origin codereview

This will either say 'up to date' or will shift the framework to the codereview branch, in which case you'll see a number of updates on your terminal/bash shell.

5. Pushing CodeReview Edits Back Up To Master

Git/Gitlab will automatically track any changes the reviewer has made, so there is no need for individual note-taking or explanation.

$ git push origin codereview

'codereview' here isn't entirely necessary, but it's safer to explicitly tell Git which branch to push changes to. Once this step is finished, send the project lead an email and they will look over any changes and pull down your code if it's an improvement.
