Form Layout

Description

Forms will follow a 25% / 75% column layout. Labels or brief descriptions on the left, and input fields on the right. At smaller screen sizes, these will collapse to a single, stacked column. form tags will be nested immediately inside the opening and closing div class="module-content" elements.

Each row of paired labels and inputs (wrapped in a <div class="form-group"> ... </div> will be followed with an <hr> tag, which will provide the consistent spacing and separation between each field.

Example

Accessibility

Proper and consistent use of name and for attributes is key.

Markup:HTML

<div class="module-content">
	<form action="" class="form-horizontal">
		<div class="form-group">
			<div class="col-md-3">
				<!-- Insert Label Here -->
			</div>
			<div class="col-md-9">
				<!-- Insert Input Here -->
			</div>
		</div>
		<hr>

		<!-- Repeat Form Groups as Needed -->
	</form>
</div>

Labels and Inputs

Description

Every label is given a class of "control-label", and every input is given a class of "form-control". Each of these matching pairs will be wrapped in a div class="form-group". Ensure that each label has a for attribute that corresponds to its input's name attribute. This match is necessary for both accessibility and back end functionality.

Example

Accessibility

Labels on the left serve as the question, allowing the decision on the right to make more sense. Consistent use of for and name attributes will further increase accessibility. Additionally, all input fields must include a tab-index attribute.

Markup:HTML

<form action="" class="form-horizontal">
	<div class="form-group">
		<div class="col-md-3">
			<label for="text" class="control-label">Example Label:</label>
		</div>
		<div class="col-md-9">
			<input type="text" name="text" class="form-control" placeholder="Example Text Input" tab-index="102">
		</div>
	</div>
	<hr>
</form>

Add File Input Button

Description

Our Add File button is built by making the default input type="file" invisible and placing a background image sprite underneath it. It is a fixed width and height, please do not attempt to resize.

Example

Accessibility

The default functionality of these inputs is retained. Only the graphic is changed. A :focus style must be applied for keyboard access.

Markup:HTML

<form action="" class="form-horizontal">
	<div class="form-group">
		<div class="col-md-3">
			<label for="addfile" class="control-label">Attach a File:</label>
		</div>
		<div class="col-md-9">
			<div class="add-file">
				<input type="file" name="addfile" class="form-control">
			</div>
		</div>
	</div>
</form>

Custom Radio Buttons and Checkboxes

Description

Form pages will all typically follow a 1/4 - 3/4 layout rule. At smaller screen sizes, they will collapse to their own row unless specifically told not to. The form tag will sit just inside the <div class="module-content"> div, and will be closed just before the <div class="module-content"> div is closed.

<div class="module-content">
  <form action="" role="form" class="form-horizontal">
    <!-- form content goes here --> 
  </form> 
</div><!-- /module-content -->

Text Inputs and Labels

Each row of a form will be wrapped in a <div class="form-group"> and will have labels on the left in a <div class="col-md-3"> and the input tag on the right inside a <div class="col-md-9">. After each form group div, insert an <hr> tag. Currently, the status classes are being applied to each row's <div class="form-group"> wrapper. These haven't been mocked up in an actual layout yet, so this structure will likely change.

<div class="form-group">
  <div class="col-md-3">
    <label for="text" class="control-label">Text - Active:</label>
  </div>
  <div class="col-md-9">
    <input type="text" class="form-control">
  </div>
</div>
<hr>

Radio Buttons and Checkboxes

These two elements are highly customized, utilize some very modern CSS pseudo-classes, and must retain this HTML structure to function properly.

<div class="form-group">
  <div class="radio">
    <input type="radio" id="radio1" name="options" value="radio1">
    <label for="radio1">Option 1</label><br>
    <input type="radio" id="radio2" name="options" value="radio2">
    <label for="radio2">Option 2</label>
  </div>
</div>
<div class="form-group">
  <div class="checkbox">
    <input type="checkbox" id="checkbox1" name="options" value="checkbox1">
    <label for="checkbox1">Checkbox Option 1</label><br>
    <input type="checkbox" id="checkbox2" name="options" value="checkbox2">
    <label for="checkbox2">Checkbox Option 2</label>
  </div>
</div>



Textarea

Textareas receive the same layout format as labels and text inputs. The RichText bar across the top is optional.

<div class="form-group"> 
  <div class="col-md-3">
    <label for="textarea">Textarea:</label>
  </div>
  <div class="col-md-9">
    <div class="richtext">
      <a href="#trigger-richtext-plugin" class="icon-richtext"></a>
    </div>

  </div>
